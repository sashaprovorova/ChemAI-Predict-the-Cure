## 1. Описание задачи

Для каждого химического соединения необходимо предсказать три целевые переменные:

| Таргет | Описание |
|---|---|
| `IC50, mM` | концентрация вещества, при которой подавляется 50% биологической активности вируса |
| `CC50, mM` | концентрация, при которой наблюдается токсичность для 50% клеток |
| `SI` | Selectivity Index, индекс селективности соединения |

Метрика соревнования — средний RMSE по трём таргетам:



```text

score = (RMSE(IC50) + RMSE(CC50) + RMSE(SI)) / 3

```

Формат итогового submission:

```text
index,IC50,CC50,SI

```

## 3. Анализ данных

В ходе EDA были выявлены основные особенности данных.

**Размеры выборок**
```text
train_df: 751 строка, 214 колонок
test_df: 250 строк, 211 колонок
```
В каждом наборе данных 211 признаков. 
Колонка index является служебным идентификатором и не используется как признак.
Наборы признаков в train и test совпали, значит одну и ту же схему обработки можно применять к обеим выборкам.

**Пропуски в данных**

Пропуски были обнаружены только в признаках, в целевых переменных пропусков нет.

В рабочей матрице признаков:
| Dataset | Количество пропусков |
|---|---:|
| train | 24 |
| test | 12 |

Строки с пропусками не удалялись, так как обучающая выборка небольшая. Для деревяных моделей пропуски обрабатывались внутри `Pipeline` с помощью `SimpleImputer(strategy="median")`. Для остальных экспериментов рабочая матрица признаков формировалась через `make_feature_matrix()`, где дополнительно предусмотрена техническая замена возможных `inf` и `-inf` на `NaN`.

**Константные и дублирующиеся признаки-колонки**

Перед моделированием из рабочего набора были исключены:
- `18` константных признаков;
- `6` полностью дублирующихся признаков-колонок.

Константные признаки имеют только одно уникальное значение и не несут полезной информации для модели.

Дублирующиеся признаки-колонки — это признаки, значения которых полностью совпадают со значениями других признаков по всем объектам обучающей выборки. Такие признаки не добавляют новой информации для модели, поэтому из каждой пары оставлялся только один признак.

|---|---|

| `fr_COO` | `fr_COO2` |
| `fr_Ar_NH` | `fr_Nhpyrrole` |
| `NumAromaticCarbocycles` | `fr_benzene` |
| `fr_nitro_arom` | `fr_nitro_arom_nonortho` |
| `fr_phenol` | `fr_phenol_noOrthoHbond` |

В результате подготовки признаков:

```text

Константных признаков: 18
Дублирующихся признаков-колонок: 6

```

**Дубликаты и типы данных**

Полных дубликатов в train обнаружено не было. 
Все колонки имеют числовые типы данных: int64 или float64. 
Нечисловых признаков нет, поэтому не требуется кодирование категориальных переменных или обработка текстовых данных. 

**Конфликтные дубликаты**

Были найдены группы строк с одинаковыми молекулярными признаками, но разными значениями таргетов.
Обнаружены повторяющиеся группы: `60` duplicate-групп, в которые входила `181` строка train. 
Из них `53` группы оказались конфликтными: одинаковому набору молекулярных признаков соответствовали разные значения хотя бы одного таргета.

Это опасно тем, что  модель видит одинаковые признаки, но разные правильные ответы. Это создаёт шум в обучении.

Были проверены разные стратегии обработки таких групп от агрегации значений и мягкой обработки дубликатов (алгоритм по удалению лишь наиболее конфликтных дубликатов описан в блокноте) до учета крайних случаев: оставления в базе признаков и полное удаление строк.

**Корреляционный анализ признаков**

На этапе EDA была проверена корреляция между молекулярными дескрипторами. Для первичного анализа использовалась Spearman-корреляция с порогом `0.8`, чтобы оценить общую степень мультиколлинеарности признаков.

Было найдено `243` пары признаков с корреляцией выше `0.8`, из них `10` пар имели корреляцию, близкую к `1.0`. Это показывает, что в данных есть заметная избыточность: часть молекулярных дескрипторов описывает близкие физико-химические или структурные свойства молекул. Например, сильно коррелируют признаки, связанные с молекулярной массой, количеством тяжёлых атомов, топологическими индексами и функциональными группами.

Корреляционный анализ в EDA использовался только как диагностический этап. Порог `0.8` не применялся для автоматического удаления признаков.
Дальше в моделировании была отдельно проверена гипотеза о корреляционном отборе признаков через Pearson correlation filter:

```text
удалять признаки с |corr| > 0.95
```
Эта гипотеза была использована в экспериментах с моделями и дала хороший результат для финального варианта.

**Выбросы**

У части признаков также наблюдаются выраженные правые хвосты. Самый заметный пример — `Ipc`: для него была отмечена сильная правосторонняя асимметрия и высокая доля выбросов. Также правосторонняя асимметрия встречается у ряда VSA/EState-дескрипторов. Поэтому при выборе моделей мы не опирались на предположение о нормальности признаков и сделали основной упор на деревья, бустинги и их ансамбли.

**Особенности целевых переменных**

Распределения IC50, CC50 и особенно SI имеют выраженную правостороннюю асимметрию.
Особенно сложным таргетом оказался SI: большинство значений небольшие, но присутствуют экстремальные выбросы. Из-за RMSE такие значения могут сильно влиять на итоговую метрику.

Также было проверено, что в train:

```text
SI = CC50 / IC50
```

То есть SI является производной величиной. 
В финальной модели SI предсказывается как отдельный таргет, как и требовалось по условию соренования.

## 4. Концепция исследования

Данные соревнования имеют несколько особенностей, из-за которых нельзя было заранее выбрать одну очевидную модель и один универсальный способ предобработки.

Во-первых, в данных есть технические и статистические сложности:
- пропуски в части молекулярных дескрипторов;
- константные признаки;
- полностью дублирующиеся признаки-колонки;
- сильная мультиколлинеарность между дескрипторами;
- скошенные распределения целевых переменных;
- особенно тяжёлый хвост у `SI`;
- конфликтные duplicate-группы, где одинаковым молекулярным признакам соответствуют разные значения таргетов.

Во-вторых, заранее не было понятно, какие модели лучше подходят для этой задачи. Молекулярные дескрипторы могут содержать нелинейные зависимости, поэтому линейные модели были использованы только как baseline и носи. Основное исследование строилось вокруг двух семейств моделей:

1. **Деревья и случайные ансамбли деревьев**, прежде всего `ExtraTreesRegressor`.
2. **Градиентные бустинги**: `CatBoost`, `LightGBM`, `XGBoost`.

**Основная идея исследования состояла не в переборе моделей, а в проверке двух гипотез.**

**Гипотеза 1. Нелинейные ансамбли должны быть сильнее линейных моделей**

Линейные модели плохо подходят для этой задачи, потому что связь между химическими дескрипторами и биологическими таргетами, скорее всего, нелинейная. Кроме того, признаки имеют разные распределения, среди них есть коррелирующие дескрипторы, а таргеты сильно скошены.

Поэтому после линейного baseline основной упор был сделан на деревья, бустинги, ансамбли и stacking. Такие модели лучше подходят для табличных химических данных, потому что:
- умеют моделировать нелинейные зависимости;
- устойчивее работают с разнотипными числовыми признаками;
- не требуют строгой нормальности признаков;
- могут учитывать взаимодействия между дескрипторами;
- ансамбли и stacking позволяют снижать variance и использовать разные источники предсказаний.

Эта гипотеза подтвердилась: линейные модели не стали конкурентными, а лучшие результаты дали ансамблевые методы.

**Гипотеза 2. Разным типам моделей нужна разная подготовка признаков**

Вторая ключевая гипотеза заключалась в том, что один и тот же набор признаков не обязан быть оптимальным для всех моделей.

Для деревьев и для бустингов были проверены две разные базы признаков и данных.

**База для деревьев**
Для `ExtraTrees` лучше всего сработала база:
- удаление наиболее плохих duplicate-групп по `CV > 0.75`;
- сохранение остальных строк без агрегации;
- отбор `top-75` признаков по важности ExtraTrees;
- ансамбль `single-output + multi-output ExtraTrees`.

Важная часть этой ветки — мягкая обработка конфликтных duplicate-групп. Сначала были проверены грубые стратегии: оставить данные как есть, агрегировать повторы или удалить все конфликтные группы. Но лучшим оказался промежуточный подход: удалять только самые нестабильные duplicate-группы по коэффициенту вариации. Так удалось убрать наиболее шумные наблюдения, но не потерять слишком много обучающих строк.

**База для бустингов**

Для бустингов лучше сработал другой подход:
- удаление экстремальных значений `SI` по порогу `q97.5`;
- удаление сильно коррелирующих признаков через Pearson filter `|corr| > 0.95`;
- итоговый набор `159` признаков;
- обучение `SI` в `log1p`-пространстве;
- `IC50` и `CC50` оставались в исходной шкале.

Эта база оказалась хуже для `ExtraTrees`, но стала полезной для бустингов, особенно для предсказания `SI`.

Таким образом, вторая гипотеза подтвердилась: для деревьев и бустингов оптимальными оказались разные схемы подготовки данных.

**Общий ход исследования**

Исследование строилось последовательно:
1. Проверили линейные baseline-модели и убедились, что они слабые.
2. Перешли к `ExtraTrees` как сильной модели на деревьях.
3. Исследовали проблему конфликтных duplicate-групп.
4. Подобрали мягкую очистку bad duplicate-групп по `CV > 0.75`.
5. Построили сильную ветку: `ExtraTrees single-output + multi-output ensemble`, т.к. multi-output модель способна: учитывать производный характер `SI`, использовать совместное распределение таргетов
и улучшать согласованность предсказаний. Вес ансамбля не фиксируется вручную, а подбирается.
6. Улучшили ветку исследования на деревьях через отбор `top-75` признаков.
7. Проверили альтернативную базу `SI <= q97.5 + Pearson-159` для деревьев и показали, что для `ExtraTrees` она хуже.
8. Проверили бустинги на лучшей деревяной базе и показали, что простая замена деревьев на бустинг не работает.
9. Проверили бустинги на альтернативной базе и увидели, что они лучше ловят `SI`, но хуже работают по `IC50` и `CC50`.
10. Сделали target-wise blend: `IC50` и `CC50` взяли от ExtraTrees, а `SI` смешали с CatBoost.
11. После этого перешли к полноценному full boosting stack: `CatBoost + LightGBM + XGBoost`, `10-fold`, `3 seeds`, `Ridge stacking`.

В итоге в исследовании появились три опорные модели:

| Этап | Модель / подход | Роль |
|---|---|---|
| ExtraTrees top-75 | `bad dups CV > 0.75 + top-75 + ExtraTrees ensemble` | сильный baseline на деревьях  |
| Target-wise blend | `ExtraTrees` для `IC50/CC50` + `CatBoost` для `SI` | показал, что бустинг полезен как источник для `SI` |
| Full boosting stack | `CatBoost + LightGBM + XGBoost + Ridge` | финальная лучшая модель |

Финальный результат дала не одна отдельная идея, а последовательное развитие гипотез: сначала была найдена сильная база для деревьев, затем показано, что бустингам нужна другая обработка данных, и после этого построен полноценный stacking бустингов.

## 5. Ограничения решения

Главное ограничение финальной модели связано с согласованностью трёх предсказываемых таргетов.

В обучающей выборке показатель `SI` фактически связан с отношением:

```text
SI = CC50 / IC50
```

Однако в финальном full boosting stack все три таргета моделируются отдельно. 
Для SI используется отдельная модельная ветка с обучением в log1p-пространстве. Поэтому в итоговом submission предсказанный SI не обязан точно совпадать с отношением predicted_CC50 / predicted_IC50.

Мы отдельно проверили это расхождение как диагностическую проверку. 
Она показала, что для части объектов независимо предсказанный SI заметно отличается от расчётного значения через IC50 и CC50.

Это означает, что финальная модель в первую очередь оптимизирует метрику соревнования, а не строго сохраняет физическую связь между тремя таргетами. 
Такой подход оказался эффективнее по public score, но его нужно учитывать как ограничение интерпретируемости решения.

Ещё одно ограничение связано с локальной валидацией: финальный boosting stack обучался и валидировался на train после удаления экстремальных значений SI выше q97.5. 
Поэтому локальный score может быть оптимистичнее результата на независимой тестовой выборке.

**Возможное направление доработки** 
Надо добавить post-processing для согласования SI с отношением CC50 / IC50, например через blend между независимо предсказанным SI и расчётным значением. 
Это может улучшить физическую согласованность, но такой шаг нужно подбирать по OOF-валидации, чтобы не ухудшить итоговую метрику.

## 8. Глобальное направление улучшения

Для дальнейшего существенного улучшения качества может потребоваться расширение признакового пространства за счёт внешних биомедицинских и химических данных.

В текущем решении используются только предоставленные молекулярные дескрипторы. Они описывают структуру соединений, но не содержат информации об экспериментальных условиях, клеточных линиях, механизмах действия, лекарственных мишенях и активности похожих молекул.

Потенциально можно обогащать данные из внешних источников, например ChEMBL, PubChem, BindingDB, DrugBank или специализированных баз по противовирусной активности. 
Особенно перспективны дополнительные данные по `IC50`, так как они чаще встречаются в открытых источниках.

Главная сложность такого подхода — сопоставимость данных. Значение `IC50` для одной и той же молекулы может сильно зависеть от клеточной линии, штамма вируса, метода измерения, времени экспозиции и лабораторного протокола. Кроме того, для корректного объединения с внешними базами нужны устойчивые идентификаторы молекул: `SMILES`, `InChI`, `InChIKey` или CAS-номер.

Поэтому внешние данные могут дать прирост качества, но это отдельная исследовательская задача: нужно надёжно сопоставить молекулы, нормализовать экспериментальные условия и проверить, действительно ли новые признаки улучшают валидацию.
